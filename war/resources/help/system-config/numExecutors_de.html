Diese Einstellung legt die Anzahl gleichzeitiger Build-Prozesse in Hudson fest. Sie beeinflußt daher die Gesamtlast, die Hudson auf einem System erzeugen kann. Ein guter Ausgangspunkt wäre die Anzahl der CPUs Ihres Systems.

Eine Erhöhung der Anzahl über diesen Wert hinaus würde zunächst die einzelnen Builds verlängern, könnte aber insgesamt den Durchsatz erhöhen, weil es den CPUs erlaubt, an einem Build zu rechnen, während ein anderer Build wegen Ein-/Ausgabeoperationen wartet.

Falls Hudson im Master/Slave-Modus betrieben wird, legt ein Wert von 0 hier fest, daß ausschließlich auf den Slaves Build-Prozesse durchgeführt werden, nie aber auf dem Master selbst.
