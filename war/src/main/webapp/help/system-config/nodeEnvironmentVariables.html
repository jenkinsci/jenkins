Environment variables defined here will be made available to every build executed on this node and will override any environment variables that have the same Name as those defined on the Configure System page.

Using the syntax $NAME or ${NAME} ( %NAME% on Windows), these variables can be used in job configurations, or from processes launched by a build.

Jenkins also supports a special syntax, BASE+EXTRA , which allows you to prepand values to an existing environment variable.

For example, if a node has PATH=/usr/bin , you could add to the standard path by defining an environment variable with the name PATH+LOCAL_BIN and value /usr/local/bin .
This would result in PATH=/usr/local/bin:/usr/bin being exported during builds executed on this node.
Multiple entries are prepended to the "base" variable according to the alphabetical order of the "extra" part of the name.

If the Value is empty or whitespace-only, it will not be added to the environment, nor will it override or unset any environment variable with the same name that may already exist (e.g. a variable defined by the system).
