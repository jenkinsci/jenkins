These key-value pairs apply for every build on every node. They can be used in Jenkins' configuration (as $key or ${key} ) and will be added to the environment for processes launched from the build.

Jenkins also supports a special syntax, BASE+EXTRA, which allows you to add multiple key-value pairs that will be prepended to an existing environment variable.

For example, if a system has PATH=/usr/bin, you could add to the standard path by defining an environment variable with the name PATH+LOCAL_BIN and value /usr/local/bin.
This would result in PATH=/usr/local/bin:/usr/bin being exported during builds. PATH+LOCAL_BIN=/usr/local/bin will also be exported.
Multiple entries are prepended to the "base" variable according to the alphabetical order of the "extra" part of the name.
