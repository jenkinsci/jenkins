Controls how Jenkins permits launching via JNLP.
Launch only from Computer detail. Require Login if security enabled.
This is the default and normal setting. In this mode, Jenkins provides the JNLP launch link on the computer view. If Security is enabled, then the link will only be available to authenticated users.
Launch from front page. Require Login if security enabled.
In this mode, Jenkins provides the JNLP launch link on both the computer view and the Build Executor Status sidepanel. If Security is enabled, then the link will only be available to authenticated users.
Launch only from Computer detail. Login never required.
In this mode, Jenkins provides the JNLP launch link on the computer view. The link will always be available whenever the slave is offline.
WARNING! In this mode security is disabled. Thus slaves can be started by anyone that can access this server. Slaves have the ability to execute arbitrary code on the master. Do not select this option unless you are fully aware of the risks
