Контролирует планирование сборок на сервере.
Загружать настолько, насколько возможно
Это обычное поведение по умолчанию. В этом режиме Jenkins свободно распоряжается ресурсами узла. Как только возникает необходимость в сборке, которая может быть выполнена этим узлом, Jenkins его использует.
Только для связанных проектов
В этом режиме Jenkins будет выполнять сборки тех проектов, которые установили его в качестве "привязанного узла". Это позволяет узлу быть зарезервированным для определенных проектов или групп проектов. Например, чтобы выполнять тесты производительности используя Jenkins, вы можете использовать эту настройку, установив количество исполнителей в 1, так что только один процесс теста производительности будет запускаться в каждый момент времени и этот процесс не будет блокироваться другими сборками, которые могут быть выполнены и на других узлах.
