Si vous cochez cette option, Jenkins conservera les javadocs de chaque build passé avec succès. Cela permet de naviguer dans les javadocs de build plus anciens, au prix d'une utilisation plus importante de l'espace disque.

Si vous laissez cette option décochée, Jenkins ne conservera que les derniers javadocs. Les anciens javadocs seront donc remplacés à chaque nouveau build qui passe avec succès.
