A scoped token can only exercise the permissions you select here, even if the owning user later gains broader access. Use this to apply the principle of least privilege to automation credentials.

Scoped (recommended): the token is restricted to the permissions you tick below. Requests that require any other permission will be rejected at the authorization layer.
Inherit all my permissions: the token behaves like a traditional API token and can do anything the owning user can. Choose this only when you need it for a legacy integration.

Note on sibling permissions: Jenkins permissions are not all transitively related. For example, Item.Configure does not imply Item.Read , so a token intended to edit jobs should usually include both. Broader permissions like Jenkins.Administer do imply everything through the impliedBy chain.
