Questo modulo mantiene sotto controllo la differenza di orario tra il master e i nodi. Mentre Jenkins stesso è in generale in grado di tollerare differenze di orario fra i sistemi, le attività legate al controllo di versione e all'accesso ai file distribuito (come NFS o condivisioni file Windows) tendono a presentare problemi strani quando i sistemi coinvolti hanno ora e data differenti.

Per mantenere gli orologi sincronizzati, si faccia riferimento a NTP.
